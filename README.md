# cv photo booth

Фотобудка, которой управляешь **жестами рук** — без мышки и клавиатуры.

Камера видит твои руки, ты «рамкой» из пальцев выбираешь кадр, щипком делаешь снимок,
а потом **собираешь из него пазл** прямо в воздухе. После нескольких фото программа
склеивает их в вертикальную **ленту как в настоящей фотобудке** и сохраняет на Рабочий стол.

> macOS (Apple Silicon) · Python · OpenCV · MediaPipe


---

## ✨ Возможности

- 🖐️ **Кадрирование двумя руками** — область снимка задаётся кончиками большого,
  указательного и среднего пальцев обеих рук (как «режиссёрская рамка»).
- 🤏 **Жест-затвор** — одновременный щипок двумя руками запускает съёмку.
- ⏱️ **Отсчёт 3 секунды** после щипка, чтобы успеть встать в позу.
- 🧩 **Пазл из снимка** — фото режется на сетку 3×3 и перемешивается; куски
  переставляются щипком (взял → перенёс → отпустил).
- 🎞️ **Фотобудочная лента** — после 3 собранных фото они склеиваются в вертикальный
  стрип с подписью и датой и сохраняются на Рабочий стол.
- 🎨 **Минималистичный UI** — полупрозрачные панели, системный шрифт SF Pro,
  адаптивная вёрстка под размер окна.

---

## 🧠 Как это работает

Одно окно OpenCV и машина состояний:

```
FRAME → COUNTDOWN → REVIEW → PUZZLE → SOLVED → (×3) → DONE
```

- Кадр с камеры всегда **зеркалится**, чтобы движения совпадали с картинкой.
- **MediaPipe Hands** отдаёт 21 точку на каждую руку; по ним считаются жесты:
  - **Щипок** — расстояние между кончиками большого и указательного пальцев,
    нормированное на размер ладони (порог не зависит от того, близко рука или далеко).
    Стабильность обеспечивают **гистерезис** (разные пороги «сжать»/«разжать») и
    **EMA-сглаживание**, чтобы жест не дребезжал.
  - **Рамка** — ограничивающий прямоугольник по кончикам пальцев обеих рук.
- Для скорости трекинг гоняется по уменьшенной копии кадра, а вся отрисовка —
  быстрыми операциями OpenCV (без тяжёлого размытия), текст с кириллицей — через PIL.

## 🚀 Установка и запуск

> Проект делался под **macOS на Apple Silicon (M2)**. Под другие ОС логика та же,
> но нюансы с доступом к камере будут отличаться.

### 1. Клонировать репозиторий

```bash
git clone https://github.com/<your-username>/gesture-photo-puzzle-booth.git
cd gesture-photo-puzzle-booth
```

### 2. Создать окружение и поставить зависимости

С [`uv`](https://github.com/astral-sh/uv) (быстро, без прав администратора):

```bash
uv venv --python 3.11
uv pip install -r requirements.txt
```

Или классическим способом:

```bash
python3.11 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3. Запустить

```bash
./.venv/bin/python main.py
```

**Управление:** `q` — выход · `r` (на финальном экране) — начать заново.

## ⚙️ Настройка

Все параметры вынесены константами в начало `main.py` — можно крутить под себя:

```python
CAMERA_INDEX = 1     # индекс камеры (встроенная камера Mac, если занят iPhone/Continuity)
PUZZLE_N     = 3      # размер сетки пазла N×N
NUM_SHOTS    = 3      # сколько фото собрать до склейки ленты
COUNTDOWN_SEC = 3     # длительность отсчёта
PINCH_ON / PINCH_OFF  # пороги щипка (гистерезис)
```

## 🩹 Возможные проблемы (macOS)

- **Камера не открывается / нет диалога разрешения.** Запускай из **Terminal**
  (ему проще выдать доступ), затем разреши камеру в *System Settings → Privacy &
  Security → Camera*. Бинарникам без нормальной подписи macOS иногда молча
  отказывает — помогает переподпись: `codesign --force -s - <путь к python>`.
- **Чёрный кадр / снимает телефон.** Это Continuity Camera: iPhone занимает
  `index 0`. Поставь `CAMERA_INDEX = 1` (встроенная камера Mac).
- **Низкий FPS.** Трекинг сыпется при плохом освещении — это не баг кода, добавь света.

`camera_check.py` — маленький скрипт-пробник: перебирает камеры и показывает,
какая отдаёт нормальную картинку.
